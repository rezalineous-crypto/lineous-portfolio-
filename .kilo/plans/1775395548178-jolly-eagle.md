# Plan: Adapt CI/CD & Docker for Vite + React

## Summary
The copied CI/CD and Docker files were designed for Next.js, but this project uses Vite + React. The following changes are needed to make the Docker build and deployment work correctly.

---

## 1. Dockerfile

**Issues:**
- References Next.js build output (`.next` folder)
- Uses `npm run build` which for Vite outputs to `dist/` not `.next`
- Uses `npm start` (Next.js production command) instead of `vite preview`
- Port 3000 is hardcoded (Vite defaults to 4173)

**Required changes:**
1. Change base image to work with Vite build
2. Update build output path from `.next` to `dist`
3. Change start command from `npm start` to `npm run preview`
4. Update exposed port from 3000 to 4173 (Vite preview default)
5. Adjust env file copying logic for Vite (use `.env.production` and `.env.development`)
6. Update build script reference to use Vite

---

## 2. docker-compose.prod.yml & docker-compose.dev.yml

**Issues:**
- Both reference port 3000 (Next.js default)
- Container images configured for Next.js

**Required changes:**
- Update port mappings from `3000` to `4173` (vite preview port)
- Optionally rename container names to reflect this is a Vite app

---

## 3. GitHub Workflow (deploy.yml)

**Issues:**
- References Next.js build commands and paths
- Port 3019/3020 mapped to internal port 3000

**Required changes:**
1. Update build command to use Vite (`npm run build`)
2. Update port mappings in deployment (3019→4173, 3020→4173)
3. Update docker-compose commands to point to correct port
4. Adjust any Next.js-specific paths

---

## 4. Environment Files (.env.production, .env.development)

**Issues:**
- Contains Next.js-specific variables (`NEXTAUTH_URL`, `NEXTAUTH_SECRET`, `NEXT_PUBLIC_*`)
- These are NextAuth.js/Next.js specific and may cause confusion in Vite

**Required changes:**
1. Review if these env vars are actually needed
2. If using NextAuth, consider updating naming or removing if not applicable
3. Vite uses `VITE_*` prefix for client-side env vars - ensure any needed variables follow this convention
4. Update variable references in code if switching from `NEXT_PUBLIC_` to `VITE_` prefix

---

## 5. Remove unused better-sqlite3 dependency

The `better-sqlite3` package in package.json is a native module that causes Docker build issues, but it's **not used anywhere in the codebase** (no imports, no usage in src/).

**Required change:**
- Remove `better-sqlite3` from package.json dependencies

---

## Implementation Order

1. Remove unused better-sqlite3 dependency from package.json
2. Update Dockerfile for Vite
3. Update docker-compose files (ports)
4. Update GitHub workflow
5. Test build locally with `docker build`
